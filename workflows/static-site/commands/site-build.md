# /site-build

全エピック（ページ）を順次構築するエントリーポイント。

## 引数
なし

## 前提
- `docs/specification.md` が作成済みであること
- `docs/design-system.md` が作成済みであること
- 現在 `dev` ブランチにいること

## 使用するサブエージェント
- `task-planner` : エピック・タスク分解
- `beads-manager` : Beadsタスク登録
- `git-manager` : 全エピック完了後のdev → mainマージ

## 処理フロー

### 1. 前提確認
- `git branch --show-current` で `dev` ブランチにいることを確認する
- `docs/specification.md` と `docs/design-system.md` の存在を確認する

### 2. 既存Beadsタスクの確認・クリーンアップ → `beads-manager`
- `bd list` で既存のopenタスクを確認する
- openタスクが存在する場合:
  - ユーザーに一覧を提示し、以下を確認する:
    - 前回のセッションで途中まで作成されたタスクか？
    - 継続すべきか、クリーンアップして最初からやり直すか？
  - クリーンアップする場合: 全openタスクを `bd close` でクローズする
  - 継続する場合: 既存タスクのステータスを確認し、未完了タスクから再開する（ステップ5へスキップ）
- openタスクが存在しない場合: そのまま次のステップに進む

### 3. エピック・タスク分解 → `task-planner`
- 仕様書を読み、エピック（ページ単位）とタスク（機能単位）に分解する
- 共通コンポーネント（ヘッダー、フッター、ナビゲーション等）は最初のエピックに含める
- 各タスクのdescriptionは `task-decomposer` スキルの `references/task-template.md` フォーマットに従う

### 4. Beadsへの登録 → `beads-manager`
- エピックを親タスクとして登録する
- 各タスクをエピックの子タスクとして登録する
- タスク間の依存関係を設定する
- 共通コンポーネントのエピックは他のエピックより先に実行されるよう依存を設定する

### 5. エピック実行順序の決定 → `task-planner`
- エピック間の依存関係に基づいて実行順序を決定する
- 共通コンポーネント → 各ページの順で実行する

### 6. エピックの順次実行
- 実行順序に従い、各エピックについて `/site-epic-execute` の処理を実行する
- 1エピックが完了したら停止せず次のエピックに進む
- いずれかのエピックで失敗した場合は `/site-failed` を実行して停止する

### 7. dev → main マージ → `git-manager`
- 全エピック完了後、`dev` ブランチ上で全テストが通ることを確認する
- `git checkout main` でmainブランチに移動する
- `git merge dev` で通常マージする（commit履歴を残す）
- `git checkout dev` でdevブランチに戻る

### 8. 完了報告
- サイト全体の構成と成果物を報告する
- `npx next build` で静的HTMLを出力し、`out/` に生成されたことを報告する
- ブラウザでの確認方法（`npx next dev` で起動）を案内する
