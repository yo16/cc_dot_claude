# knowledge

AIエージェントが参照する技術制約・過去の教訓を蓄積するディレクトリ。

## 目的

- エージェント定義（agents/）は汎用的に保ちつつ、技術固有の制約や過去のトラブル事例をエージェントに伝える
- 同じ失敗を繰り返さないよう、教訓をナレッジとして蓄積する

## ファイル形式

各ファイルは以下の構造で記述する:

```markdown
# <タイトル>

## 適用条件
- どのような技術スタック・状況でこの制約が該当するか

## 制約・注意点
- 何をしてはいけないか、何に気をつけるべきか

## 対策
- 正しい対応方法

## 背景
- なぜこの制約があるのか（過去のトラブル等）
```

## プロジェクトへの導入手順

1. プロジェクトの `.claude/` にワークフローをコピーする際、`agents/` や `commands/` と同じ階層に `knowledge/` フォルダを作成する
2. このディレクトリから、そのプロジェクトの技術スタックに該当するファイルだけを選んでコピーする
3. エージェントが自動的に `knowledge/` 内のファイルを参照する

```
.claude/
  commands/           ← コマンド定義
  agents/             ← エージェント定義
  knowledge/          ← ここにコピー（該当するものだけ）
    nextjs-edge-runtime.md
  settings.json
```
